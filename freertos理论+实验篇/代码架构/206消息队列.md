3.
响应按键的打印处理实验

首先，是写入，（调用入队函数）
按键按下，打印字符串，
并将数组首地址sc_aucButtonstring_Upi赋值给指针c_pucsendstring。
xQueueSendFromISR(PrintQueueHandle, &c_pucsendstring , 0);

&c_pucsendstring ;是一个二级指针，&c_pucsendstring 是这个指针变量自身的地址
sc_aucButtonstring_Upi[]是数组
sc_aucButtonstring_Upi与
&sc_aucButtonstring_Upi 是数组的首地址

如果，传入&sc_aucButtonstring_Upi，
那么出队函数，接收到的就是数组的首字符，
而非指针变量。
我们出队函数，接收到的指针变量c_pucsendstring，返回给另一个指针变量。


注意：sc_aucButtonstring_Upi[]是字符串的数组，不能是英文中文结合，不然会printf乱码


出队：
打印函数，
void vPrintButtonString()
{
    uint8_t *p;   // 声明一个指针变量 p
    xQueueReceive(PrintQueueHandle, &p, PortMAX_DELAY);
    printf("%s", p);
}

将返回值，赋值给指针变量的指针，&P。
xQueueReceive它会从队列里读出一个指针值，写入以&p这个地址开始的p的size 大小的字节里。
p 被赋值为队列中传来的那个字符串的首地址
打印，printf("%s",p)

在任务中，调用这个打印函数


再考虑一中情况，
如果打印时间比较长，
且按键按下速度很快，来不及读取完，
队列就被填满了。
所以，有一些我们是读取不到的。
当队列大小从1-某个有限值都会出现这种情况


const 类型 * const 指针名：指针本身和指向的内容都不可修改

const 类型 * 指针名（const 在 * 左边）：指向常量的指针，指针变量本身可以修改，可以让它指向不同的内存地址；

类型 * const 指针名（const 在 * 右边）：指针常量，指针变量本身是只读的，定义后不能再修改指向；
